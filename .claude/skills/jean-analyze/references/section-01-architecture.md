# Section 01: 아키텍처 분석 (`01-architecture.md`)

`/jean:analyze` 본문에서 이 reference 를 따라 아키텍처 리포트를 생성한다.

**생성 조건**: 항상 (모든 프로젝트에 해당)

## 분석 순서

1. **디렉토리 구조 스캔** — 소스 루트 하위 주요 디렉토리를 Glob 으로 수집한다.
2. **레이어 구조 파악** — Discovery 결과의 `detected_stack` 기반으로 각 레이어의 역할을 정리한다.
   - API 레이어: `controllers/`, `routes/`, `handlers/`, `views/`, `pages/` 등
   - 비즈니스 레이어: `services/`, `usecases/`, `application/` 등
   - 데이터 레이어: `repositories/`, `models/`, `entities/` 등
   - 인프라 레이어: `clients/`, `adapters/`, `infrastructure/` 등
3. **DI / 의존성 구조** — DI 컨테이너 파일이 있으면 Read → Provider 관계 정리. 없으면 import 패턴으로 의존 관계를 추론한다.
4. **설정 관리** — 설정 파일 Read → 환경변수 / 설정 항목을 표로 정리한다. 값은 변수명만 기재 (보안).
5. **예외 / 에러 처리** — `exceptions/` 또는 `errors/` 디렉토리가 있으면 클래스 계층을 정리한다.
6. **미들웨어 / 인증** — auth 관련 파일이 있으면 인증 흐름을 정리한다.
7. **앱 진입점** — `main.py`, `app.py`, `index.ts`, `main.go` 등 Read → 초기화 흐름을 기재한다.
