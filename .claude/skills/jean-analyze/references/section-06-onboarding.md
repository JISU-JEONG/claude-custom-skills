# Section 06: 신입 온보딩 가이드 (`06-onboarding.md`)

**생성 조건**: 항상 (모든 프로젝트에 해당)

## 분석 순서

1. **프로젝트 소개** — `CLAUDE.md` 또는 `README.md` Read → 한 줄 소개 추출. 없으면 코드에서 추정한다.
2. **환경 세팅** — 패키지 매니저 감지 결과로 설치 명령어 구성.
   - Python: `poetry install` / `uv sync` / `pip install -r requirements.txt`
   - Node.js: `npm install` / `pnpm install` / `yarn install`
   - Go: `go mod download`
   - Rust: `cargo build`
   - Java: `./mvnw install` / `./gradlew build`
3. **환경변수** — `.env*` 파일 Read → 필요한 변수 목록 (값은 마스킹).
4. **DB 마이그레이션 (해당 시)** — alembic, prisma migrate, django migrate 등.
5. **서버 실행 명령** — 진입점 파일에서 추출 또는 `package.json` scripts.
6. **읽기 순서** — 프로젝트 구조 기반으로 핵심 파일 10개 이내 선정.
   - 항상 포함: 진입점, 설정, 라우팅 / API 정의
   - 있으면 포함: DI 설정, 핵심 비즈니스 로직, 데이터 모델
7. **핵심 비즈니스 흐름** — API 엔드포인트 중 가장 복잡한 2~3개를 워크스루.
8. **자주 하는 작업 가이드** — 기존 코드 패턴 분석 → "새 기능 추가" step-by-step.
9. **디버깅 팁** — 로깅 설정, 테스트 실행 방법, 환경 전환 방법.
