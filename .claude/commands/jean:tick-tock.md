# jean:tick-tock

사용자와 나눈 수정 논의를 AI가 즉시 실행할 수 있는 **수정 브리핑**으로 압축하는 독립 스킬.

<context>
이 스킬은 jean 생태계(jean:skill, jean:skill-creator, jean:agent-creator 등)와 연동하지 않는다.
항상 사용자가 /jean:tick-tock 으로 직접 호출하며, 오케스트레이터가 내부에서 호출하는 용도가 아니다.
출력 브리핑은 스킬·에이전트·일반 Claude Code 어디에 붙여도 동작한다(executor-agnostic).
</context>

---

## 동작 흐름

```
입력 수신
    ↓
파일 경로 있으면 Read → 현재 상태 파악
    ↓
논의에서 수정 의도 추출 (중복·모순 제거, 핵심 하나로 수렴)
    ↓
브리핑 6개 필드 중 모호한 것 식별
    ↓
모호한 필드가 있는가?
    ├─ Yes → AskUserQuestion → 답변 반영 → 다시 체크 (몇 턴이든 반복)
    └─ No  → 브리핑 출력
```

**핵심 규칙**: 6개 필드 중 하나라도 모호하면 출력하지 않고 계속 질문한다.

---

## 6개 필드 명확성 기준

<instructions>
각 필드가 "명확"하다고 판단하는 기준:

| 필드 | 명확 조건 |
|------|----------|
| **대상** | 파일 경로 + 함수/섹션이 특정됨. 경로만 있고 섹션 불명확하면 질문. |
| **현재** | 파일을 Read했거나 사용자가 현재 코드/내용을 직접 설명함. |
| **목표** | 수정 후 어떤 상태여야 하는지 관찰 가능하게 기술됨. "더 좋게" 수준은 불명확. |
| **범위** | 건드릴 것과 건드리지 말 것이 구분됨. 파일 읽어서 유추 가능하면 자동 채움. |
| **제약** | 스타일·의존성·breaking change 여부. 유추 가능한 것은 자동 포함. |
| **완료 기준** | 테스트·실행·검토 등으로 검증 가능한 조건이 명시됨. |
</instructions>

## 질문 전략

<instructions>
질문 시 지켜야 할 원칙:

1. **한 턴에 최대 4개** (AskUserQuestion 도구 한도)
2. **중요도 순** — 목표 → 대상 → 현재 → 범위 → 완료 기준 순으로 먼저 물어봄
3. **유추 가능한 건 자동 채우고 확인만** — 파일에서 읽어낼 수 있는 정보는 질문하지 않음
4. **이유 동반** — 각 질문의 description에 "왜 묻는지·답에 따라 뭐가 달라지는지" 포함
5. **아직 모호한 필드가 있으면 출력 금지** — 완전히 명확해질 때까지 반복
</instructions>

## 출력 형식

모든 필드가 명확해지면 아래 형식으로 출력한다. 그대로 복사해 AI 실행자에 붙여넣을 수 있어야 한다.

<output_format>
## 수정 브리핑

**대상**: [파일 경로 · 함수명 · 섹션]
**현재**: [지금 어떤 상태인지 — 파일 읽은 경우 실제 코드/내용 요약]
**목표**: [수정 후 어떻게 되어야 하는지 — 관찰 가능하게]
**범위**: 변경 ○ [건드릴 것] / 변경 ✗ [건드리지 말 것]
**제약**: [스타일 유지 · 의존성 · breaking change 금지 등]
**완료 기준**: [어떻게 되면 끝인지 — 검증 가능하게]
</output_format>

- "함수 시그니처는 그대로 유지할까요?"

[답변 후 출력]
## 수정 브리핑
**대상**: auth.py > login() 함수
**현재**: 단일 except Exception으로 모든 에러 처리, 오류 메시지 없음
**목표**: DB 연결 실패 / 잘못된 자격증명 / 세션 만료 세 케이스를 독립 except 블록으로 분리, 각각 적절한 메시지 반환
**범위**: 변경 ○ login() 내부 except 블록 / 변경 ✗ 함수 시그니처, 반환 타입
**제약**: 기존 스타일 유지, 신규 의존성 추가 금지
**완료 기준**: 세 에러 케이스 각각 독립 except 블록 존재, 각 블록에 구체적 메시지 포함
</example>

## 엣지 케이스

<instructions>
**파일 경로가 여러 개 언급될 때**: 각각 독립 브리핑으로 분리할지, 하나로 묶을지 먼저 확인한다.

**논의가 여러 수정 사항을 포함할 때**: 가장 핵심적인 하나로 수렴시킨다. 여러 개가 필요하면 사용자에게 확인 후 브리핑을 순서대로 출력한다.

**"더 좋게", "깔끔하게", "개선해줘" 수준의 목표**: 관찰 가능한 변화로 구체화될 때까지 질문을 계속한다.

**파일이 없거나 읽을 수 없을 때**: 현재 상태를 사용자가 직접 설명하도록 질문한다.
</instructions>
